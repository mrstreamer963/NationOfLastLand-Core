Да, использование Rust для создания DLL-плагинов для Unity зачастую предпочтительнее, чем Golang, особенно когда нужно вызывать нативный код в Unity. Rust предлагает более зрелые и удобные средства для компиляции в динамическую библиотеку (DLL) с экспортом функций, которые можно импортировать в Unity через PInvoke (DllImport в C#).​

Преимущества Rust по сравнению с Golang для таких задач:

Хорошая поддержка создания динамических библиотек (DLL, .so) с экспортом C-совместимых функций.

Более прямой контроль над памятью и возможностями маршализации, что важно для взаимодействия с Unity.

Существуют готовые проекты, такие как runity, облегчающие интеграцию Rust-кода в Unity, включая поддержку базовых Unity-типов (например, GameObject, Transform) и автоматическую загрузку библиотек.​

Rust-плагины можно легче обновлять и расширять без необходимости постоянного перезапуска Unity (если организовать загрузку с разными именами DLL).​

Большое сообщество и инструментарий для нативной разработки, что упрощает отладку и сопровождение.

В то время как Golang можно использовать для создания DLL, ограничения и сложность взаимодействия с Unity делают Rust более предпочтительным выбором при разработке нативных расширений для Unity.

Итог: Для интеграции через DLL в Unity Rust обычно лучше подходит, чем Golang, благодаря лучшей поддержке нативных вызовов, управлению памятью и доступным инструментам интеграции
